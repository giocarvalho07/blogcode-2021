Modelagem conceitual de dados - Técnicas de análise de dados

Postado em 2 de Maio de 2021 - Giovanni de Carvalho
Modelagem de dados Sql

Introdução a técnica Ajudar na investigação os processos: 1- Entender melhor as situações problemas; 2- Estruturar o banco de dados de forma adequada; 3- Ferramenta para o desenvolvimento de sistemas; 4- Permitir o compartilhamento de dados; 5- Organizar a integração dos sistemas e compartilhamento de informações; 6- Ferramenta para administração de dados; 7- Simplificar a visão da organização perante as informações; 8- Evitar problemas de redundância de dados;

Análise top down

Também conhecido como método de refinamento sucessivo. Esse método baseia-se na estratégia de dividir para conquistar o resultado. O problema é dividido em subproblemas (problemas menores) que podem ser mais fáceis de resolver. Com isso, você divide a situação-problema em situações de menor complexidade e soluciona uma de cada vez até atingir o objetivo.

Análise de requisitos

Como vimos nos exemplos, o mesmo objeto pode gerar diversos pontos de vista. A mesma coisa acontece com a análise de sistemas. Para termos uma análise mais segura, temos de analisar o sistema a partir de pontos diferentes exemplos; Usuário; Negocio; Processo; O banco de dados é um conjunto de objetos que relacionam entre si de forma organizada. Os objetos que compõem o banco de dados podem ser, tabelas (de cliente, fornecedores, produtos, estoque), consultas, relatórios, etc
